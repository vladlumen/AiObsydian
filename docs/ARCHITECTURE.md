🛠️ Действующий Стек Моделей (ИИ-Ядро)
Текст и Логика (Основная): Hermes 3 (8b). Запущена через плоский эндпоинт /api/generate при temperature: 0.0. Идеально следует Obsidian-разметке, игнорирует внутренние рассуждения CoT и выдает готовый результат RAG-инжеста в пределах 10–12 секунд.

Зрение (OCR и Скриншоты): Llama 3.2 Vision. Выделенная модель Ollama для точного распознавания текста на изображениях, схемах и интерфейсах Unity/Roblox.

Речь в текст (STT): Faster-Whisper (large-v3). Локальный изолированный инференс на CPU (device="cpu", compute_type="int8") для полной экономии VRAM.

Векторные эмбеддинги: Nomic Embed Text. Инференс векторов переведен со старого плоского метода на выделенный эндпоинт /api/embed API Ollama, что сократило время получения Query-вектора до 0.9–1.3 секунд для базы знаний LanceDB.

🧠 Логика Обработки Задач (Защита VRAM)
Для исключения перегрузки видеопамяти RTX 3090 (24 ГБ) вся обработка событий переведена на строго последовательный конвейер:

TaskQueue работает в 1 активный фоновый воркер. Одновременный параллельный инференс тяжелых нейросетей аппаратно запрещен на уровне логики очереди.

Взаимодействие между Orchestrator и внешним интерфейсом Telegram Bot изолировано с помощью ленивых (внутренних) импортов, что полностью решает проблему циклических зависимостей Python при старте ядра.

Контроль изменений Obsidian Vault осуществляется по принципу проекта Khoj: через отслеживание mtime и хэширование файлов в SQLite (file_registry), предотвращая холостой перерасчет векторов.

Воркер TaskQueue выполняет автоматическую зачистку папки data/temp_media/ в блоках finally сразу после завершения или падения задачи.

🔧 Архитектурные ограничения / Паттерны взаимодействия
Orchestrator как тонкий диспетчер: Класс Orchestrator является исключительно легковесным маршрутизатором событий и не содержит внутри себя тяжелой логики инференса или парсинга данных. Все вычисления делегируются специализированным агентам.

Правило немедленной передачи в TaskQueue: Любая обработка событий из шины, требующая инференса нейросетей или парсинга данных, должна немедленно передаваться в TaskQueue для соответствующего Агента.

Двусторонний RAG-триггер: Поисковый префикс ? валидируется и считывается Orchestrator как в начале строки, так и на конце сообщения, с последующим полным очищением Query-текста в RAGAgent.

Свободный контекстный инференс по содержанию: Исключена избыточная агрессия галлюцинаторного фильтра в системном промпте поиска (модели разрешено сопоставлять синонимы и анализировать содержание заметки, даже если исходный .md заголовок усечен лимитами ОС). Имя исходного файла принудительно инъецируется в тело контекста чанка для жесткой связки.
