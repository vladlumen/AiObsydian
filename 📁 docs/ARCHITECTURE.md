# Архитектура и Структура Проекта (v2.3)

**Правила Хранения Данных:**
- ВЕСЬ код, базы данных (LanceDB, SQLite) и временные медиа (аудио, фото из TG) живут в Linux `/home/vladislav/projects/agent_project/`.
- ТОЛЬКО финальные `.md` файлы пишутся в Windows `/mnt/c/Users/.../ObsidianVault/`.

## 🛠️ Действующий Стек Моделей (ИИ-Ядро)
- **Текст и Логика (Основная):** `Qwen 3.5 (9b)` (Instruct-режим с обязательным флагом `think=False` в API для удержания тайминга генерации в пределах 35 секунд).
- **Зрение (OCR и Скриншоты):** `Llama 3.2 Vision` (Используется как выделенная модель для точного распознавания текста на изображениях, схемах и интерфейсах Unity/Roblox).
- **Речь в текст (STT):** `Faster-Whisper (large-v3)` (Локальный инференс через фоновые потоки).
- **Векторные эмбеддинги:** `Nomic Embed Text` (Генерация векторов для базы знаний LanceDB).

## 🧠 Логика Обработки Задач (Защита VRAM)
Для исключения перегрузки видеопамяти RTX 3090 (24 ГБ) вся обработка событий переведена на **строго последовательный конвейер**:
1. `TaskQueue` работает в 1 воркер. Одновременный инференс тяжелых моделей запрещен.
2. Взаимодействие между `Orchestrator` и внешним интерфейсом `Telegram Bot` изолировано с помощью **ленивых (внутренних) импортов**, что полностью решает проблему циклических зависимостей Python при старте ядра.
