## 🛠️ Действующий Стек Моделей (ИИ-Ядро)
* **Текст и Логика (Основная):** `Hermes 3 (8b)`. Запущена через плоский эндпоинт `/api/generate` при `temperature: 0.0`. Идеально следует Obsidian-разметке, игнорирует внутренние рассуждения CoT и выдает готовый результат RAG-инжеста в пределах 10–12 секунд.
* **Зрение (OCR и Скриншоты):** `Llama 3.2 Vision`. Выделенная модель Ollama для точного распознавания текста на изображениях, схемах и интерфейсах Unity/Roblox.
* **Речь в текст (STT):** `Faster-Whisper (large-v3)`. Локальный изолированный инференс на CPU (`device="cpu"`, `compute_type="int8"`) для полной экономии VRAM.
* **Векторные эмбеддинги:** `Nomic Embed Text`. Генерация векторов для базы знаний LanceDB.

---

## 🧠 Логика Обработки Задач (Защита VRAM)
Для исключения перегрузки видеопамяти RTX 3090 (24 ГБ) вся обработка событий переведена на строго последовательный конвейер:
1. `TaskQueue` работает в 1 активный фоновый воркер. Одновременный параллельный инференс тяжелых нейросетей аппаратно запрещен на уровне логики очереди.
2. Взаимодействие между `Orchestrator` и внешним интерфейсом Telegram Bot изолировано с помощью ленивых (внутренних) импортов, что полностью решает проблему циклических зависимостей Python при старте ядра.
3. Контроль изменений Obsidian Vault осуществляется по принципу проекта Khoj: через отслеживание `mtime` и хэширование файлов в SQLite (`file_registry`), предотвращая холостой перерасчет векторов.
4. Воркер `TaskQueue` выполняет автоматическую зачистку папки `data/temp_media/` в блоках `finally` сразу после завершения или падения задачи.
