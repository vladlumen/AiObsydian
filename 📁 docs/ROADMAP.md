# План разработки Personal AI Assistant (v2.3)

**Стратегия:** Двигаемся снизу вверх. Сначала строим надежный транспорт и управление ресурсами, затем подключаем "мозги", и только в конце — бизнес-логику агентов и память.

---

## 🏗️ Phase 1: Foundation (Инфраструктура) — [ВЫПОЛНЕНО]
**Цель:** Создать неубиваемое ядро, которое умеет ставить задачи в очередь и безопасно работать в рамках 24GB VRAM.
- [x] Создать `event_bus.py` (Шина сообщений + Dataclasses).
- [x] Создать `registry.py` (Конфиг требований к памяти).
- [x] Оптимизировать утилизацию VRAM: перевести `task_queue.py` в строго последовательный однопоточный режим (замена динамического `vram_scheduler.py`).
- [x] Создать `orchestrator.py` (Подписка на эвенты + устранение циклических импортов бота через ленивую загрузку).

---

## 🧠 Phase 2: Cognitive Layer (Подключение "Органов чувств") — [ВЫПОЛНЕНО]
**Цель:** Оживить бота, чтобы он начал слышать, видеть и писать, грамотно распределяя GPU.
- [x] Интеграция `stt_service.py` (Faster-Whisper + фикс драйверов cuBLAS в Linux).
- [x] Интеграция `llm_service.py` (Переход на флагманскую `Qwen 3.5 (9b)` + жесткое подавление CoT через `think=False`).
- [x] Интеграция `vision_service.py` (Выделенная `Llama3.2-Vision` для точного OCR кода и скриншотов движков).
- [x] Подключение Telegram: `handlers.py` (Прием voice/text/photo с защитой от спам-реакций).
- [x] Добавить `telemetry.py` (Замер времени ответа и утилизации VRAM).

---

## 💾 Phase 3: The "Second Brain" (Хранилище и Память) — [ВЫПОЛНЕНО]
**Цель:** Научить агента безопасно писать в Obsidian и "вспоминать" старые записи через векторную базу.
- [x] Реализовать `obsidian_writer.py` (Атомарная запись `.md` по мосту в `/mnt/c/...` + обрезка имен до 60 символов).
- [x] Создать `storage/vector_store.py` (Интеграция LanceDB локально в Linux для хранения эмбеддингов).
- [x] Создать `cognitive/memory/semantic.py` (Логика RAG: автоматическое сохранение и индексация векторов через `Nomic Embed`).
- [x] Создать `cognitive/memory/conversation.py` (Краткосрочная память сессии).
