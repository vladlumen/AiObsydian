# ROADMAP.md
# План разработки Personal AI Assistant (v1.0)

**Стратегия:** Двигаемся снизу вверх. Сначала строим надежный транспорт и управление ресурсами (чтобы не убить GPU), затем подключаем "мозги", и только в конце — бизнес-логику агентов.

---

## 🏗️ Phase 1: Foundation (Инфраструктура)
**Цель:** Создать неубиваемое ядро, которое умеет ставить задачи в очередь и безопасно жонглировать моделями в рамках 24GB VRAM.

- [ ] Создать `infrastructure/event_bus.py` (Шина сообщений + Dataclasses для эвентов).
- [ ] Создать `models/registry.py` (Конфиг весов и требований по памяти).
- [ ] Создать `infrastructure/vram_scheduler.py` (Семафор, загрузка/выгрузка моделей, защита от OOM).
- [ ] Создать `infrastructure/task_queue.py` (Асинхронная очередь для защиты от спама).
- [ ] Создать базовый `core/orchestrator.py` (Подписка на тестовые эвенты).
> **Критерий завершения:** Скрипт может принять 10 одновременных "фейковых" голосовых сообщений и по очереди обработать их, корректно блокируя и освобождая VRAM.

---

## 🧠 Phase 2: Cognitive Layer (Подключение "Органов чувств")
**Цель:** Оживить бота, чтобы он начал слышать, понимать и говорить (в тексте), используя железо.

- [ ] Интеграция `services/stt_service.py` (Faster-Whisper). Связка с VRAM Scheduler.
- [ ] Интеграция `services/llm_service.py` (Hermes 3 8B через Ollama/Llama.cpp).
- [ ] Подключение Telegram: `interfaces/telegram/handlers.py` (Прием voice/text и отправка в EventBus).
- [ ] Создать `cognitive/memory/conversation.py` (Краткосрочная память сессии в SQLite).
> **Критерий завершения:** Ты отправляешь голосовое в Telegram -> Бот его расшифровывает -> LLM генерирует осмысленный ответ -> Бот присылает текст. (Без падений по памяти).

---

## 💾 Phase 3: The "Second Brain" (Хранилище и Obsidian)
**Цель:** Научить агента безопасно работать с твоей файловой системой и долгосрочной памятью.

- [ ] Создать `storage/vector_store.py` (Настройка LanceDB для хранения эмбеддингов).
- [ ] Создать `cognitive/memory/semantic.py` (Логика RAG: поиск похожих заметок перед ответом).
- [ ] Реализовать `agents/obsidian_writer.py` (Атомарная запись `.md` файлов по мосту в `/mnt/c/...`).
> **Критерий завершения:** Бот может сохранить твою мысль из Telegram прямо в папку `00_Inbox` на диске Windows в правильном Markdown-формате с тегами.

---

## 🤖 Phase 4: Autonomous Behaviors (Агенты и Зрение)
**Цель:** Сделать систему самостоятельной и мультимодальной.

- [ ] Добавить Vision: `services/vision_service.py` (LLaVA). Распознавание фото из Telegram.
- [ ] Написать `agents/clerk_agent.py` (Сортировщик: анализ `00_Inbox` и раскидывание по папкам).
- [ ] Добавить обработчик документов `parsers/pdf_parser.py` и `docx_parser.py`.
- [ ] Настроить планировщик (Cron/Asyncio) для ежедневного запуска Clerk Agent.
> **Критерий завершения:** Ты кидаешь боту скриншот кода или PDF-статью -> он ее анализирует -> сам решает, что это относится к работе -> создает красивую заметку в папке `work`.

---

## 🚀 Phase 5: Polish & Observability (Релиз)
**Цель:** Сделать систему прозрачной для дебага и приятной в использовании.

- [ ] Добавить `infrastructure/telemetry.py` (Замер времени ответа и потребления VRAM).
- [ ] Обработка краевых сценариев (пользователь удалил сообщение, пока оно было в очереди).
- [ ] Рефакторинг системных промптов в `config.py`.
