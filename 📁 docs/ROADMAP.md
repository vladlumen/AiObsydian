# План разработки Personal AI Assistant (v1.0)

**Стратегия:** Двигаемся снизу вверх. Сначала строим надежный транспорт и управление ресурсами, затем подключаем "мозги", и только в конце — бизнес-логику агентов и память.

---

## 🏗️ Phase 1: Foundation (Инфраструктура) — [ВЫПОЛНЕНО]
**Цель:** Создать неубиваемое ядро, которое умеет ставить задачи в очередь и безопасно жонглировать моделями в рамках 24GB VRAM.
- [x] Создать `event_bus.py` (Шина сообщений + Dataclasses).
- [x] Создать `registry.py` (Конфиг весов: Hermes, Whisper, Llama-Vision).
- [x] Создать `vram_scheduler.py` (Семафор, защита от OOM).
- [x] Создать `task_queue.py` (Асинхронная очередь воркеров).
- [x] Создать `orchestrator.py` (Подписка на эвенты).

---

## 🧠 Phase 2: Cognitive Layer (Подключение "Органов чувств") — [ВЫПОЛНЕНО]
**Цель:** Оживить бота, чтобы он начал слышать, видеть и писать, грамотно распределяя GPU.
- [x] Интеграция `stt_service.py` (Faster-Whisper + фикс драйверов cuBLAS в Linux).
- [x] Интеграция `llm_service.py` (Hermes 3 8B через Ollama).
- [x] Интеграция `vision_service.py` (Llama3.2-Vision + сжатие Pillow + хард-промпты).
- [x] Подключение Telegram: `handlers.py` (Прием voice/text/photo с защитой от спам-реакций).
- [x] Добавить `telemetry.py` (Замер времени ответа и утилизации VRAM).

---

## 💾 Phase 3: The "Second Brain" (Хранилище и Память) — [В ПРОЦЕССЕ]
**Цель:** Научить агента безопасно писать в Obsidian и "вспоминать" старые записи.
- [x] Реализовать `obsidian_writer.py` (Атомарная запись `.md` по мосту в `/mnt/c/...` + обрезка имен).
- [ ] Создать `storage/vector_store.py` (Настройка LanceDB для хранения эмбеддингов).
- [ ] Создать `cognitive/memory/semantic.py` (Логика RAG: поиск похожих заметок перед ответом).
- [ ] Создать `cognitive/memory/conversation.py` (Краткосрочная память сессии).

---

## 🤖 Phase 4: Autonomous Behaviors (Агенты) — [ПЛАН]
**Цель:** Сделать систему самостоятельной.
- [ ] Написать `agents/clerk_agent.py` (Сортировщик: анализ `00_Inbox` и раскидывание по папкам).
- [ ] Добавить обработчик документов `parsers/pdf_parser.py` и `docx_parser.py`.
- [ ] Настроить планировщик (Cron/Asyncio) для ежедневного запуска Clerk Agent по расписанию.
